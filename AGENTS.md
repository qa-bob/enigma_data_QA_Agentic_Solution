# Enigma Data QA Suite — Agent Instructions

Shared instructions for all AI coding agents (Claude Code, GitHub Copilot, Cursor, Devin, etc.)
working in this repository. This file is imported by `CLAUDE.md` and read by any agent that
understands `AGENTS.md`.

---

## Project Purpose

Build and maintain a comprehensive GUI, functional, and regression test suite for
**[enigmadata.net](https://enigmadata.net/)** — a B2B data-privacy SaaS website.

Tests must cover every discoverable feature of the site **without** requiring account creation,
login, or actual form submission. Target URL and feature flags are in `site.config.json`.

---

## Architecture: Page Object Model + OOP

Every page or major section of the site has its own **TypeScript class** in `src/pages/`.

```
BasePage (abstract base class)
  └── HomePage extends BasePage
  └── NavigationPage extends BasePage
  └── ContactFormPage extends BasePage
  └── <PageName>Page extends BasePage   ← one class per discovered page
```

### Page Object Rules

- All page classes **extend `BasePage`** from `./base.page`
- Locators are `readonly Locator` properties declared on the class — not inline in methods
- Methods represent **user actions** (click, fill, navigate) — never assertions
- **No `expect()` calls inside page objects** — assertions belong exclusively in spec files
- Constructor signature: `constructor(page: Page, config: SiteConfig)`

### Test Rules

- Always import `{ test, expect }` from `@fixtures/site.fixture` — never directly from `@playwright/test`
- Tag every test with **at least one** of: `@smoke`, `@navigation`, `@forms`, `@functional`, `@visual`, `@responsive`
- **Never hardcode URLs** — use `baseURL` from the Playwright config (which reads `site.config.json`)
- **Never submit forms** — test field interactions and validation only
- **Never create accounts** or enter real credentials
- Do not use `page.waitForTimeout()` — use `waitForSelector` or rely on Playwright auto-waiting
- Tests must be independent — no shared mutable state across tests

### TypeScript Rules

- Strict mode is enabled in `tsconfig.json` — no implicit `any`
- All page object properties must be explicitly typed (`readonly Locator`, `readonly string`, etc.)
- Run `npx tsc --noEmit` to verify clean compilation before finalizing changes
- Path aliases: `@pages/*`, `@fixtures/*`, `@utils/*`, `@types/*` (configured in `tsconfig.json`)

## Test Category Reference

| Tag | When to use |
|-----|-------------|
| `@smoke` | Site loads, title present, no JS console errors |
| `@navigation` | Nav links, routing, menu open/close, breadcrumbs |
| `@forms` | Form fields, placeholder text, validation messages, accessibility |
| `@functional` | Business features: CTAs, pricing, accordions, modals, video |
| `@visual` | Screenshot regression with `toHaveScreenshot()` |
| `@responsive` | Viewport-specific layout — mobile 390px, tablet 768px, desktop 1280px |

---

## Writing or Updating Tests — Workflow

1. Read `site.config.json` to get the URL and feature flags
2. Fetch the live page with `WebFetch` before writing any selectors — use real HTML, not guesses
3. Add or update the relevant page object class in `src/pages/`
4. Write the spec file using the page object — no raw `page.locator()` calls in test bodies
5. Run `npx tsc --noEmit` to verify clean TypeScript
6. Tag every test correctly

## Do Not

- Submit any form
- Create accounts or log in (unless `auth.required: true` in `site.config.json`)
- Hardcode the base URL in tests — read it from `baseURL` / `siteConfig.url`
- Put `expect()` assertions inside page object methods
- Use `page.waitForTimeout()` — use explicit selectors or Playwright auto-waiting
- Use the `any` type without an explicit justification comment
- Add `console.log` calls to test or page-object files
- Edit `playwright.config.ts` project definitions without verifying the viewports match `site.config.json`
